# fd

## 문제 설명

> file descriptor

- fd.c

## 풀이

### 분석

1.  **입력값 처리**: 실행 인자 `argv[1]`을 `atoi`로 정수화한 뒤 `0x1234`를 뺀다. 이 결과값을 `fd` 변수에 저장한다.
```c
    int fd = atoi( argv[1] ) - 0x1234;
```
2.  **데이터 읽기**: `read(fd, buf, 32)` 함수를 통해 `fd`로부터 32바이트를 읽어 `buf`에 저장한다.
3.  **조건 비교**: `buf`에 저장된 문자열이 `"LETMEWIN\n"`과 일치하면 플래그를 출력하는 코드가 실행된다.

### 취약점

**리눅스 표준 파일 디스크립터(Standard File Descriptor)** 의 정의를 이용하는 문제이다.

- **0**: 표준 입력 (stdin)
- **1**: 표준 출력 (stdout)
- **2**: 표준 에러 (stderr)

만약 `fd` 변수의 값을 **0**으로 만들 수 있다면, `read(0, buf, 32)`가 호출되어 프로그램은 파일이 아닌 **사용자의 키보드 입력**으로부터 데이터를 읽게 된다.

### 익스플로잇

1.  **계산**: `fd`를 0으로 만들기 위해 `argv[1]`에 들어갈 값을 계산한다.
    - `X - 0x1234 = 0` 이 되어야 하므로, `X = 0x1234`.
    - 16진수 `0x1234`를 10진수로 변환하면 **4660**.

2.  **실행**: 계산한 값 4660을 인자로 전달하여 프로그램을 실행한다.
    ```bash
    $ ./fd 4660
    ```

3.  **입력**: 프로그램이 실행된 후 입력 대기 상태가 되면 `LETMEWIN`을 입력한다.

## 배운 점

- 리눅스 시스템에서 파일 디스크립터 0, 1, 2번이 가지는 의미를 이해한다.
- `read()` 함수가 파일뿐만 아니라 표준 입력 스트림으로부터 데이터를 읽어올 수 있음을 확인한다.
