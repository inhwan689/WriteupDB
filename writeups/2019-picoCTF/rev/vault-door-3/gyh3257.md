# Vault Door 3

## 문제 설명

> Daedalus Corporation has upgraded their vault! Can you break into this new model, agent?
> The source code for this vault is here: `VaultDoor3.java`.

`checkPassword`가 입력 32바이트를 일련의 인덱스 변환을 통해 `buffer[32]`로 옮긴 뒤, 고정 문자열과 비교한다. 단순 평문 비교가 아닌 **순열 변환**이 들어가 있어 인덱스 매핑을 역산해야 한다.

## 풀이

### 분석

```java
public boolean checkPassword(String password) {
    if (password.length() != 32) return false;
    char[] buffer = new char[32];
    int i;
    for (i = 0; i < 8; i++)         buffer[i] = password.charAt(i);
    for (;     i < 16; i++)         buffer[i] = password.charAt(23 - i);
    for (;     i < 32; i += 2)      buffer[i] = password.charAt(46 - i);
    for (i = 31; i >= 17; i -= 2)   buffer[i] = password.charAt(i);
    return new String(buffer).equals("jU5t_a_sna_3lpm16g84c_u_4_m0r846");
}
```

루프 별 매핑:

| 루프 | i 범위 | `buffer[i]` ← `password[src]` |
|---|---|---|
| 1 | 0..7 | `password[i]`  (identity) |
| 2 | 8..15 | `password[23 - i]`  → password 인덱스 15→8 (역순) |
| 3 | 16,18,20,…,30 | `password[46 - i]`  → password 인덱스 30,28,…,16 (역순, 짝수만) |
| 4 | 31,29,27,…,17 | `password[i]`  (identity, 홀수만) |

각 루프가 buffer의 어느 인덱스에 password의 어느 인덱스를 쓰는지 정확히 알면, 비교 대상 문자열 `target`에 대해 `password[src_idx] = target[buf_idx]` 로 역산 가능하다.

### 풀이 과정

1. `target = "jU5t_a_sna_3lpm16g84c_u_4_m0r846"` 를 그대로 사용.
2. 4개 루프를 그대로 시뮬레이션하되, 쓰기 방향만 뒤집어 password 배열을 채운다.
3. 32바이트 모두 채워졌는지 확인 후 `picoCTF{...}` 출력.

```python
target = 'jU5t_a_sna_3lpm16g84c_u_4_m0r846'
p = [''] * 32
for i in range(8):
    p[i] = target[i]
for i in range(8, 16):
    p[23 - i] = target[i]
for i in range(16, 32, 2):
    p[46 - i] = target[i]
for i in range(31, 16, -2):
    p[i] = target[i]
print(f"picoCTF{{{''.join(p)}}}")
```

복원된 buffer를 같은 변환에 다시 넣어 target과 일치하는지 검증해 정확성을 보장한다.

## 배운 점

- "입력을 셔플 후 고정 문자열과 비교"는 결국 **순열의 역**을 구하는 문제다. 각 루프 변수가 어떤 인덱스에 무엇을 쓰는지 표로 정리하면 즉시 풀린다.
- VaultDoor 시리즈에서 흔히 보이는 패턴이며, 더 큰 바이너리에서도 동일 구조가 등장한다(특히 obfuscator가 분기 비교 순서를 셔플할 때).
- 자동 솔버를 만들 때 핵심은 "buffer-write 표현식의 source-index"를 추출하는 정적 분석 — 이게 가능하면 패턴 1/3 류 문제는 사실상 자동화 가능.
