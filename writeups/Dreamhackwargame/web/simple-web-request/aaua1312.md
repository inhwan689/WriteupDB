## 풀이

### 분석

문제 파일의 압축을 풀어서 보니 flag.html, step1.html, step2.hetml, app.py, flag.txt(의미없었음) 등의 파일이 있었다. app.py를 열어보니 /, /step1, step2, /flag 엔드포인터가 있었는데 결과적으로 step1에서 특정 파라미터를 입력하면 step2로 넘어가고 step2에서 또 다른 특정 파라미터를 입력하면 /flag 에서 flag가 나타나는 방식이었다.

입력값 검증 취약(신뢰되지 않은 입력을 그대로 사용한다.)
